Executive Summary

Somalite is a fully integrated, cloud-based School Management System (SMS) purpose-built for secondary and tertiary institutions. It replaces paper-based processes, fragmented spreadsheets, and disconnected software tools with a single, unified platform that covers every aspect of school operations — from student admissions and academic records, to fee collection, payroll, timetabling, and parent communication.

Built on modern, enterprise-grade technology and designed specifically for the African school environment, Somalite empowers school administrators, teachers, bursars, directors of studies, parents, and students through dedicated role-based portals — each tailored to the exact needs of that user.

This document outlines the full feature set, portal structure, benefits, and implementation approach of the Somalite platform.

1. Overview & Platform Architecture

Somalite is a multi-tenant, web-based platform accessible on any device with a modern browser — desktop, tablet, or smartphone. There is no software to install and no server infrastructure to maintain. Schools are onboarded within hours and data is securely stored in the cloud with automatic backups.

Core Technology Stack

  • Next.js 15 Modern, fast web framework with server-side rendering

2. School Administration Portal

The School Admin Portal is the command centre of the platform. School administrators have full visibility and control over every module. The dashboard provides live snapshots of student enrollment, fee collection status, pending approvals, and recent activity across all departments.

Key Capabilities

  • School Profile Management School name, logo, address, motto, contact details, and brand colours for portal theming
  • Multi-Year Academic Setup Create and manage multiple academic years and terms, set active year, configure term dates
  • Class & Stream Management Define class templates (e.g. S.1, S.2), create class years per academic year, and manage streams (e.g. S.2A, S.2B) with class head assignments
  • User Account Management Create and manage accounts for teachers, staff, bursars, parents, and students with role-based access control
  • Role Assignment & Permissions Assign multiple roles to staff members, control what each user can see and do within the system
  • Activity Audit Log Full audit trail of all sensitive operations including mark changes, fee adjustments, and user modifications
  • School Colour Theming Each school's portal reflects its own brand colours automatically applied throughout the interface

3. Academic Management

Somalite provides a complete academic structure management system, handling everything from class creation through to student enrollment and subject assignment.

Academic Year & Term Management

  • Create and archive academic years (e.g. 2025/2026)
  • Define terms per year (Term 1, Term 2, Term 3) with start and end dates
  • Mark one term as active — the system uses this for all mark entry, attendance, and fee workflows
  • Historical data preserved across all years for auditing and reporting

Classes & Streams

  • Define class templates with levels (e.g. Senior 1 through Senior 6)
  • Create class years that link class templates to academic years
  • Create multiple streams per class year (e.g. S.3A, S.3B, S.3C, S.3 Sciences)
  • Assign a class head teacher to each stream

Subject Management

  • Create a school-wide subject library (subject name, code, department)
  • Assign subjects to specific class years with paper configurations (Paper 1, Paper 2, Paper 3)
  • Assign subjects to streams and link subject teachers
  • Configure assessment settings per subject: mark weights, AOI settings, grading scales
  • Full paper management for O-Level and A-Level examination configurations

Student Enrollment

  • Enroll students into specific streams and terms
  • Subject-level enrollment to track which students take which subjects
  • Bulk student promotions at end of year (promote entire classes with one action)
  • Transfer students between streams with full transfer history records
  • Enrollment status tracking: Active, Completed, Transferred, Withdrawn

Student Profiles

  • Comprehensive student record: name, admission number, date of birth, gender, religion, nationality, blood group, medical conditions, photo
  • Parent/guardian linkage with contact details
  • Full enrollment history across academic years
  • Academic results history, attendance history, fee account summary
  • Project works portfolio (up to 10 items per student)

4. Examination & Continuous Assessment

Somalite implements a multi-stage marks management workflow designed to ensure accuracy, accountability, and proper sign-off at every level before results are published.

Mark Entry by Teachers

  • Teachers enter marks per student per subject paper directly from their portal
  • Three exam types supported: BOT (Beginning of Term), MTE (Mid-Term Exam), EOT (End of Term)
  • Marks saved as DRAFT, then submitted for class teacher review
  • Mark entry form shows each student's name, admission number, and current mark
  • Configurable maximum marks per paper

Three-Stage Approval Workflow

StageActorActionResulting Status
Stage 1Subject TeacherEnters marks and submits for reviewSUBMITTED
Stage 2Class Head TeacherReviews and approves or rejects with commentsCLASS_TEACHER_APPROVED / REJECTED
Stage 3Director of StudiesFinal approval — locks results permanentlyAPPROVED

All stage changes are timestamped and attributed to the acting user. Rejected marks must be corrected and resubmitted by the teacher.

Areas of Integration (AOI) — Continuous Assessment

  • AOI supports practical, project-based, and continuous assessment scoring
  • Class teachers define AOI topics per subject (with topic number, name, max points)
  • Teachers enter scores per student per AOI topic
  • Competence level, generic skills assessment, and remarks fields per topic
  • Same three-stage approval workflow applies to AOI scores
  • AOI scores contribute to final subject marks per configured weight

Grading Configuration

  • Admin-configurable grade boundaries per subject or school-wide (A, B, C, D, F with mark ranges)
  • Separate AOI grading scales configurable per class
  • Division calculation for O-Level (Division I, II, III, IV, Fail based on aggregate)
  • Points-based grade calculation for A-Level
  • Mark audit log records every change with before/after values

5. Report Card System

Somalite generates professional, printable report cards automatically from approved marks. Three report card types are supported, each with full school branding.

Report Card Types

  • End of Term Report Card Complete term results with subject marks, grades, positions, division, teacher remarks, and head teacher comment
  • Mid-Term Report Card Interim results for parent communication at mid-term
  • AOI Assessment Report Dedicated report showing AOI topics, scores, competence levels, generic skills, and remarks per subject

Report Card Features

  • School logo, name, address, and motto displayed on every card
  • Student photo, name, class, stream, admission number, and term
  • Subject-by-subject breakdown with paper marks, totals, grades, and positions
  • Class average and student position in class
  • Overall aggregate, division (O-Level), or points tally (A-Level)
  • Teacher remarks per subject and head teacher general comment
  • Signature lines for class teacher and head teacher
  • School colour-themed borders and headers
  • Toggle controls: admin/DOS can choose which fields appear on printed cards
  • Print All button generates PDF-ready output for the entire class at once

6. Student Attendance

Daily student attendance is tracked at the stream level by the assigned class head teacher. The system enforces one attendance record per student per calendar day, preventing duplicate entries.

Teacher Attendance Marking

  • Class head teachers see only their assigned stream(s) in the attendance portal
  • Date picker defaults to today; teachers cannot mark future dates
  • Four status options per student: Present, Absent, Late, Excused
  • Optional notes field per student (e.g. "doctor's appointment")
  • Mark All shortcuts Set all students to one status in a single click
  • Summary count tiles show Present / Absent / Late / Excused totals at a glance
  • Re-submitting on the same day updates existing records (no duplicates)
  • Badge confirms when a day's attendance has already been marked

Admin Attendance Management

  • Admin can view and edit attendance for any stream on any date
  • Admin can correct or override attendance submitted by teachers

Attendance Reports

  • Term-level attendance summary per stream: select stream + term and click Generate
  • Class-wide attendance rate with colour-coded progress bar (green ≥80%, amber 60–79%, red <60%)
  • Per-student breakdown: total days, present, late, absent, excused, and attendance rate
  • Students ranked by attendance rate to identify at-risk learners
  • Per-student attendance history with date, status, notes, and the teacher who marked it

7. Finance & Fee Management

Somalite includes a complete, end-to-end school finance module. Every financial transaction — from invoice generation to payment receipt — is tracked, attributed, and reportable.

Fee Structure Setup

  • Define fee categories (Tuition, Development Fund, Boarding, Meals, Uniforms, etc.)
  • Create fee structures per class level, term, and boarding status
  • Configure installment plans with due dates and amounts
  • Set penalty rules for late payment (flat fee or percentage, with grace period)

Invoicing

  • Auto-generate invoices for all students in a class/term with one action
  • Invoice PDF generation with school branding and payment details
  • Invoice status tracking: Draft, Sent, Partially Paid, Paid, Overdue

Payment Recording

  • Record payments against specific invoices (cash, bank transfer, mobile money)
  • Generate payment receipts with school logo and receipt number — styled with school brand colours
  • Partial payments supported with outstanding balance tracking
  • Carry-forward of credits from one term to the next

Bursaries & Scholarships

  • Define bursary/scholarship programmes with eligibility and funding amounts
  • Assign bursaries to individual students — allocations automatically deducted from fee accounts
  • Bursary reports showing total awarded per term

Student Fee Accounts

  • Each student has a persistent fee account showing full payment history
  • Running balance (amount owed or in credit) visible to admin, bursar, and parent
  • All invoices, payments, bursaries, and penalties linked to the account

Financial Reports

  • Daily, weekly, monthly, and term-level collections reports
  • Outstanding fees report — list of students with unpaid balances
  • Revenue breakdown by fee category
  • Expense summary with category breakdown and vendor details

9. Timetable Generation

Somalite includes an AI-powered timetable engine that automatically generates clash-free, optimised school timetables. Manual scheduling for large schools is error-prone and time-consuming; Somalite solves this in minutes.

Configuration

  • Define school day structure: number of periods, break times, day start and end times
  • Configure class-specific day structures if different classes have different schedules
  • Set subjects' periods-per-week requirements per class
  • Mark teacher availability slots (blocked periods, part-time constraints)

AI Generation Engine

  • Genetic algorithm-based generation ensures no teacher double-booking
  • No classroom or stream conflicts
  • Respects teacher blocked slots and availability windows
  • Multiple timetable versions can be generated and compared
  • Conflict detection and reporting — identifies any unresolvable constraints

Timetable Publishing

  • Publish the approved version to make it live for all users
  • Teachers see their personal weekly timetable in their portal
  • Students see their class timetable
  • PDF export for printing and display on notice boards

18. Security & Multi-Tenancy

Data Security

  • Each school's data is fully isolated — no data is ever shared between schools
  • All connections encrypted via HTTPS/TLS
  • Secure session-based authentication with NextAuth
  • Role-based access: users can only see data relevant to their role
  • Password hashing and secure credential storage
  • Full audit log of all sensitive data changes

Multi-Tenancy

  • Each school is a completely separate tenant with its own data, branding, and configuration
  • Schools access their portal via a unique school slug URL (e.g. somalite.com/school/st-marys)
  • Super-admin dashboard for platform management across all schools

Reliability

  • Cloud-hosted with 99.9% uptime SLA
  • Automatic daily database backups
  • File storage on Cloudflare R2 with redundancy
  • No hardware procurement or server maintenance required by the school
